# Renewlet v0.1.9

Renewlet v0.1.9 continues polishing family cost sharing ([zhiyingzzhou/renewlet#34](https://github.com/zhiyingzzhou/renewlet/pull/34), thanks [@Shy0o](https://github.com/Shy0o)). Custom splits now better match real recovery workflows: enter the amount each member should cover, and Renewlet will show the member total, your share, and the recoverable amount consistently.

---

## Read Before Upgrading

Docker users do not need any manual migration when upgrading from v0.1.8.

Cloudflare users should use the existing deploy flow; this release includes the data migration required for family cost sharing.

Cloudflare one-click deploy users should still upgrade from the generated repository by running `Sync Renewlet Upstream`. Do not click the one-click deploy button again. If an older generated repository does not have this entry yet, add it once from the [Cloudflare deploy documentation](https://github.com/zhiyingzzhou/renewlet/blob/main/docs/cloudflare-workers-deploy.md#existing-one-click-deploy-users).

---

## Improvements

- Custom splits no longer require member amounts to add up to the subscription price, which better fits shared subscriptions where you track real recovery amounts.
- Cost-sharing summaries more clearly separate member total, your share, and recoverable amount.
- Member management is now separate from the main subscription form, so longer member lists no longer stretch the form.
- Cross-currency member amounts now stay consistent across the subscription form, cards, details, statistics, and CSV export.
- Cloudflare deploy documentation now states the one-click and manual upgrade paths more clearly.

---

## Bug Fixes

- Fixed inaccurate recoverable amounts when custom split amounts did not match the subscription price.
- Fixed member management interactions that could unexpectedly close the subscription dialog or lose focus.
