# Renewlet v0.1.0

Renewlet v0.1.0 is the first stable release. It brings subscriptions, renewal reminders, spending views, calendar feeds, data migration, and both deployment targets into a versioned release line.

## Read Before Upgrading

If you deployed Renewlet before 2026-06-04, open the old version before upgrading and use Export subscriptions -> JSON to save `subscriptions.json`. After upgrading, import that old JSON from Import data.

`subscriptions.json` is only for subscription-data migration. It is not a full instance backup. Keep a backup of `.env`, `docker-compose.yml`, and `data/` before upgrading.

Production Docker deployments should pin a concrete version tag such as `0.1.0`. The `latest` tag only moves on stable releases; release candidates only use the concrete RC tag and the `rc` tag.

If you are still on the old Docker layout, run `docker compose pull && docker compose up -d` once to bridge to the new layout. Later stable releases can use the version badge at the top of Renewlet to open System Update and run the in-app update.

## New Features

- Stable release line: add the `0.1.0` SemVer version, versioned Docker images, GitHub Release assets, `checksums.txt`, and a Docker deployment package.
- Docker self-hosting: keep the React frontend, Go/PocketBase backend, SQLite data, and static assets in one container, with production deployments pinned to `0.1.0`.
- Cloudflare Workers deployment: support Workers Static Assets, Worker API, D1, R2, and Cron Triggers without the Go/PocketBase server.
- Core subscription fields: track price, currency, billing cycle, renewal date, status, category, payment method, tags, website, notes, Logos, and pinned items.
- Renewal calendar: add global private ICS feeds and per-subscription feeds for calendar apps ([#9](https://github.com/zhiyingzzhou/renewlet/issues/9), thanks [@Chuyiyue](https://github.com/Chuyiyue)).
- Data migration: support Renewlet import/export and migration from Wallos JSON, backup.zip, and SQLite databases.
- Notification channels: support Telegram, Notifyx, Webhook, WeCom Bot, SMTP email, and Bark.

## Improvements

- Mobile web layout covers the subscription list, bottom navigation, filter drawers, settings, and notification configuration.
- Multi-currency conversion supports Exchange API and FloatRates, with fallback rates when remote providers are unavailable.
- Logo selection supports upload, image URL, built-in icon source search, and favicon suggestions.
- The top version badge opens System Update; Docker release images support in-app binary updates, while Cloudflare and source deployments show their manual upgrade paths.
- Docker Hub Overview now uses generated content so README images and documentation links resolve correctly on Docker Hub.

---

## Bug Fixes

- Fix settings navigation highlighting when scrolling through sections.
- Fix notification history showing raw `email` instead of the localized channel name.
- Fix Cloudflare `/system/restart` semantics so unsupported restarts are not shown as successful.
- Fix Cloudflare ICS tests to unfold lines before assertions and catch calendar feed regressions.
- Replace scattered frontend `console.error` paths with unified error reporting and an Error Boundary.

---

## Security and Reliability

- Add Go and Cloudflare `/api/app/ready` endpoints to check DB/D1 readiness.
- Harden outbound notification URLs by rejecting unsafe protocols, userinfo, localhost, private networks, and IP literals.
- Move Go notification scheduler logs to structured logging for easier delivery and retry diagnosis.
- Run the website Docker image as a non-root user, move Nginx to port 8080, and add security/cache headers.
