# Renewlet v0.1.0

Renewlet v0.1.0 是第一个稳定版发布，把订阅记录、续费提醒、统计、日历订阅、数据迁移和双运行面部署整理成一条可长期维护的发布线。

[English ->](./v0.1.0-en.md)

---

## 升级前请先读

如果你是在 2026-06-04 之前部署的 Renewlet，升级前请先打开旧版本的订阅页，使用“导出订阅”里的 JSON 导出功能保存 `subscriptions.json`。升级后可通过“导入数据”导入这份旧版 JSON。

`subscriptions.json` 只用于订阅数据迁移，不是完整实例备份。升级前仍建议同时备份 `.env`、`docker-compose.yml` 和 `data/`。

生产 Docker 部署建议固定到 `0.1.0` 这类具体版本标签；`latest` 只随稳定版移动，候选版只使用具体 RC tag 和 `rc` tag。

如果你还在使用旧 Docker 布局，需要先执行一次 `docker compose pull && docker compose up -d` 切到新布局；之后的稳定版可点击页面顶部的版本号入口，打开“系统更新”执行页面内更新。

## 新增功能

- 稳定发布线：新增 `0.1.0` SemVer 版本、固定版本 Docker 镜像、GitHub Release 附件、`checksums.txt` 和 Docker 部署包。
- Docker 自托管：React 前端、Go/PocketBase 后端、SQLite 数据和静态资源保持单容器运行，生产环境可固定到 `0.1.0`。
- Cloudflare Workers 部署：支持 Workers Static Assets、Worker API、D1、R2 和 Cron Triggers 的无 Go/PocketBase 部署路径。
- 订阅核心能力：支持订阅价格、币种、周期、续费日、状态、分类、付款方式、标签、网站、备注、Logo 和置顶。
- 续费日历：支持全局私有 ICS Feed 和单个订阅 Feed，可在系统日历应用中订阅（[#9](https://github.com/zhiyingzzhou/renewlet/issues/9)，感谢 [@Chuyiyue](https://github.com/Chuyiyue)）。
- 数据迁移：支持 Renewlet 导入导出，并可迁移 Wallos JSON、backup.zip 和 SQLite 数据库。
- 通知渠道：支持 Telegram、Notifyx、Webhook、企业微信机器人、SMTP 邮件和 Bark。

## 优化改进

- H5 移动端适配订阅列表、底部导航、筛选抽屉、设置页和通知配置。
- 多币种换算支持 Exchange API 和 FloatRates，并在远端不可用时使用备用汇率。
- Logo 选择支持上传、图片链接、内置图标来源搜索和 favicon 候选。
- 页面顶部版本号入口可打开“系统更新”；Docker release 镜像支持页面内二进制更新，Cloudflare 和源码部署展示对应的手动升级路径。
- Docker Hub Overview 使用专用生成内容，修复相对图片和文档链接在 Docker Hub 页面不显示的问题。

---

## Bug 修复

- 修复设置页目录激活态在滚动时可能停留在错误 section 的问题。
- 修复通知历史里渠道名称显示 raw `email` 的问题。
- 修复 Cloudflare `/system/restart` 的语义，避免把不支持的重启操作展示成成功。
- 修复 Cloudflare ICS 测试对 folded line 的断言方式，避免日历内容回归漏测。
- 修复前端分散 `console.error` 路径，统一到错误上报和 Error Boundary。

---

## 安全与可靠性

- 新增 Go 和 Cloudflare `/api/app/ready`，用于检查 DB/D1 连接是否可用。
- 通知外发 URL 增加防护，拒绝不安全的协议、userinfo、本机地址、私网地址和 IP literal。
- Go 通知调度日志改为结构化日志，便于定位发送和重试问题。
- 官网 Docker 镜像改为非 root 运行，Nginx 监听 8080，并补齐安全头和缓存头。
